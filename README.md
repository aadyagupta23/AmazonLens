# Amazon Lens

Amazon.in clone with built-in purchase intelligence. Surfaces the data Amazon already has — price history, trust scoring, real-owner insights, and sustainability metrics — natively inside the standard shopping flow.

Built for **Amazon HackOn Season 6**.

---

## Features

### TrustLens™
On every product page: a 12-month price history chart, a colour-coded trust score (0–100), fake discount detection, and a buy-now-or-wait recommendation. Click the info icon to open the **TrustCard** — a breakdown across 5 dimensions: Review Authenticity, Return Rate, Warranty Claims, Seller Reliability, and Price Stability.

Two status tiers:
- **VERIFIED** (≥ 75) — green badge, trustworthy product
- **TRUSTED** (≥ 90) — blue badge, highest confidence

### WitnessPanel™
Chat with AI personas of verified owners. Not reviews, not customer service — actual people who own the product, responding in character with city-specific context (Bengaluru hard water, Mumbai humidity). Powered by Groq (llama3-70b).

### Amazon Sense™
Predicts when you're about to run out of regularly-ordered items. Shows as a popup on the homepage and a "Soon" tab in the cart. Backed by MongoDB — usage patterns persist across sessions.

### Green Choice™ Sustainability
- **Green Choice badge** on product cards for products from companies with eco score > 80
- **Eco Mode** — toggle from the navbar, search results banner, or sustainability dashboard. Highlights eco-friendly products across the shopping flow.
- **Sustainability Dashboard** (`/sustainability`) — fetches live data from the server on every visit. Shows your eco profile, top eco purchases, and company sustainability rankings with real scores computed from carbon targets, renewable energy %, recyclable packaging, supply chain scores, and product longevity.
- Company eco labels: Climate Leader (≥90), Eco Advanced (≥80), Eco Conscious (≥70), Eco Aware (≥60), Progressing (≥50), Developing (≥40)

### AI Bundles
- **Shopping Bundles page** (`/bundles`) — curated collections built around real shopping goals
- **AI-personalised bundles** — generated by Groq based on your actual order history. Two bundles created per session, stored in localStorage. Shown in a "For You" section on the bundles page and on the homepage via "Continue Your Journey."
- Bundle detail page shows all included products, per-item reasons (AI-generated), suggested add-ons, and an Add Entire Setup to Cart button.

### Co-Planner
Shared shopping lists. Create a plan, invite others via a token link, add items, assign them, vote, split expenses, and track purchase status. Full comment thread and activity timeline per plan.

### Smart Search
Natural language search with TrustLens-aware ranking. Detects bundle queries ("home theatre setup under 40000") and surfaces a Bundle Card above results.

### Returns Assistant
AI-powered return suggestions based on order history, reason classification, and policy matching. Out-of-stock products never appear as return swap suggestions.

## Setup

```bash
# 1. Clone and install all dependencies
git clone <repo-url>
cd amazon-lens
npm run install:all

# 2. Edit server/.env with your values:
#    PORT, MONGODB_URI, JWT_SECRET, GROQ_API_KEY

# 3. Start both client and server
npm run dev
```

- Client → http://localhost:5173
- Server → http://localhost:5001
- Health check → http://localhost:5001/api/health

## Demo Path

### 1. Homepage
- Products load in Deal of the Day and Recommended for You sections
- After 3 seconds, the **Amazon Sense™ popup** slides up — Nescafé is due for reorder
- "Continue Your Journey" section shows AI-personalised bundles based on your order history
- Click **"Try it on Sony TV →"** in the TrustLens banner to jump to the product page

### 2. Search
Type `home theatre setup under 40000` in the search bar.

- A **Bundle Card** appears at the top: Sony TV + JBL Soundbar + Fire TV Stick
- Individual product cards show TrustLens badges and **Green Choice** badges where applicable
- If Eco Mode is on, an eco banner shows with a "Turn off" toggle

### 3. Product Page — Sony Bravia 55" 4K TV (`/dp/p001`)

| What you see | What it means |
|---|---|
| Orange **71/100** trust badge | Mixed — some concerns detected |
| Red **"Fake Discount Detected"** banner | Price has been "discounted" for 10 of 12 months |
| Orange **"Consider Waiting"** box | Prime Day in 8 days — historically drops 18–22% |
| **12-month price history chart** | Red spike zones at inflated MRP months |
| **Suspicious Reviews** section | Bot-pattern reviews flagged and collapsed |
| **Sustainability Panel** | Shows if eco score > 80 — carbon footprint, recyclability, ethical sourcing |

### 4. WitnessPanel
Scroll down on the Sony TV page.

- Three owner cards: Arjun M. (Bengaluru), Priya S. (Mumbai), Rahul K. (Delhi)
- Click **Chat** on any card and ask city-specific questions
- Try: *"how does it handle hard water?"* — Arjun references Bengaluru water quality

### 5. Eco Mode + Sustainability Dashboard
- Click the **Eco Mode** toggle in the navbar to enable
- Visit `/sustainability` to see your sustainability profile with live server data
- Toggle eco mode on/off from the dashboard header

### 6. Bundles (`/bundles`)
- Place an order first (add items to cart and checkout)
- The "For You" section shows 2 AI-generated bundles built around your purchase
- Click any bundle to see all products, per-item reasons, and suggested add-ons

### 7. Cart and Sense
- Add products to cart — cart badge updates in the navbar
- Open cart — items show with TrustLens score badges
- Click the **"Soon"** tab — Nescafé appears as "Due today" with a Reorder button

### 8. Auth
- `/signup` — create an account saved to MongoDB with hashed password
- `/login` — sign back in
- Without login, the app uses demo user "Arjun Kumar" so the demo path never breaks

## API Reference

All endpoints served from `http://localhost:5001`.

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/health` | Server status |
| GET | `/api/products` | All products (`?category=` `?limit=`) |
| GET | `/api/products/:id` | Single product |
| POST | `/api/products/search` | Search with bundle detection — `{ query }` |
| GET | `/api/companies` | All company eco data |
| GET | `/api/companies/stats` | Eco stats summary |
| GET | `/api/companies/:id` | Single company |
| GET | `/api/bundles` | Curated static bundles |
| POST | `/api/bundles/ai` | AI-personalised bundles — `{ recentOrder, olderOrders, allPurchasedIds, history }` |
| GET | `/api/witness/:productId` | Owner personas |
| POST | `/api/witness/chat` | Groq chat — `{ productId, witnessId, message, history }` |
| GET | `/api/sense/predictions` | Reorder predictions |
| POST | `/api/sense/event` | Track usage event |
| GET | `/api/sense/profile` | User DNA profile |
| POST | `/api/sense/seller-trust` | Seller trust analysis |
| GET | `/api/sense/risk/:productId` | Risk profile for a product |
| GET | `/api/sense/recommendations` | Personalised recommendations |
| POST | `/api/smart-search` | Natural language smart search |
| POST | `/api/returns/suggestions` | Return swap suggestions |
| GET | `/api/co-planner/my-plans` | User's co-planner lists |
| POST | `/api/co-planner/create` | Create a new shared plan |
| POST | `/api/auth/signup` | Create account — `{ name, email, password }` |
| POST | `/api/auth/login` | Sign in — `{ email, password }` |

## Known Limitations

- Filter checkboxes in search results are UI-only (no backend filtering)
- Cart and orders use localStorage — do not persist to the backend
- WitnessPanel personas exist only for p001 (Sony TV); other products get generated responses
- Co-Planner plans are stored in a flat JSON file on the server (not MongoDB)
- Checkout flow is a stub — does not process real payments
